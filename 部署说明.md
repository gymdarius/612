# Version1 离线部署说明

本文用于将当前 `Version1` 代码带到离线环境后，快速理解算法链路、模型推理输入输出和部署注意事项。

## 1. 总体算法链路

当前工程是“规则算法生成标签 + 神经网络拟合规则 + ONNX 离线推理”的结构。

主链路包含三个算法：

1. 态势评估 `SituationAssessment`
   - 输入：导弹与目标状态。
   - 输出：导弹-目标态势评分矩阵 `situation_matrix`，shape 为 `[M, T]`。
   - 规则融合项：接近速度、距离、瞄准/视场、导弹能量。

2. 威胁评估 `ThreatAssessment`
   - 输入：友方飞机与目标状态。
   - 输出：友机-目标威胁矩阵 `threat_matrix`，shape 为 `[A, T]`；以及目标威胁权重 `target_threat_weights`，shape 为 `[T]`。
   - 规则融合项：LOS 指向、距离、速度、高度差。
   - 注意：当前代码将 `position[1]` 作为高度轴。

3. 目标分配 `TargetAllocation`
   - 输入：态势矩阵和目标威胁权重。
   - 核心公式：

```text
value = 0.99 * situation_matrix + 0.01 * target_threat_weights
normalized = row_softmax(value)
target_index = argmax(normalized, axis=target)
```

当前分配策略是“每枚导弹独立选择最优目标”，没有“一目标只能分配一枚导弹”的全局约束。

## 2. 固定输入格式

全局固定 shape 来自 `shape_config.py`：

```text
M_MAX = 4
A_MAX = 4
T_MAX = 8
FEATURE_DIM = 6
```

每个实体输入 6 维：

```text
[x, y, z, vx, vy, vz]
```

ONNX 推理只允许 batch 维动态，实体数量维度必须固定：

```text
X_missile:  [B, 4, 6]
X_aircraft: [B, 4, 6]
X_target:   [B, 8, 6]
mask_M:     [B, 4]
mask_A:     [B, 4]
mask_T:     [B, 8]
```

真实实体放在数组前部，不足的位置补 0；对应 mask 中真实实体为 `1`，padding 为 `0`。

## 3. 三个训练模型的推理输入输出

### 3.1 态势评估模型 `situation_model.onnx`

输入：

```text
X_missile: [B, 4, 6]
X_target:  [B, 8, 6]
mask_M:    [B, 4]
mask_T:    [B, 8]
```

输出：

```text
S_pred:        [B, 4, 8]
joint_mask_MT: [B, 4, 8]
```

含义：

- `S_pred[b, i, j]` 表示第 `b` 个样本中，第 `i` 枚导弹对第 `j` 个目标的态势评分。
- `joint_mask_MT[b, i, j]` 表示该导弹-目标 pair 是否真实有效。

### 3.2 威胁评估模型 `threat_model.onnx`

输入：

```text
X_aircraft: [B, 4, 6]
X_target:   [B, 8, 6]
mask_A:     [B, 4]
mask_T:     [B, 8]
```

输出：

```text
Threat_pred:   [B, 4, 8]
ThreatW_pred:  [B, 8]
joint_mask_AT: [B, 4, 8]
```

含义：

- `Threat_pred[b, i, j]` 表示第 `i` 架友方飞机面对第 `j` 个目标的威胁评分。
- `ThreatW_pred[b, j]` 是按目标聚合并归一化后的目标威胁权重。
- `joint_mask_AT[b, i, j]` 表示该友机-目标 pair 是否真实有效。

### 3.3 目标分配模型 `allocation_model.onnx`

输入：

```text
X_missile:  [B, 4, 6]
X_aircraft: [B, 4, 6]
X_target:   [B, 8, 6]
mask_M:     [B, 4]
mask_A:     [B, 4]
mask_T:     [B, 8]
```

输出：

```text
Alloc_logits:      [B, 4, 8]
allocation_index:  [B, 4]
allocation_mask:   [B, 4]
```

含义：

- `Alloc_logits[b, i, j]` 是第 `i` 枚导弹选择第 `j` 个目标的原始 logits。
- `allocation_index[b, i]` 是第 `i` 枚导弹选择的目标下标，范围为 `0..7`。
- `allocation_mask[b, i]` 表示第 `i` 个导弹槽位是否是真实导弹。

业务使用时必须只读取 `allocation_mask == 1` 的导弹位置。padding 导弹槽位即使输出了目标编号，也不能作为真实分配结果。

## 4. 端到端 ONNX 推荐接口

离线部署推荐优先使用：

```text
version1_aircombat_pipeline.onnx
```

输入与目标分配模型相同：

```text
X_missile, X_aircraft, X_target, mask_M, mask_A, mask_T
```

输出：

```text
S_pred:           [B, 4, 8]
Threat_pred:      [B, 4, 8]
ThreatW_pred:     [B, 8]
allocation_index: [B, 4]
allocation_mask:  [B, 4]
```

这样一次推理即可得到态势、威胁和最终目标分配结果，适合离线集成和调试。

## 6. 离线部署注意事项

1. ONNX 只允许 batch 维动态，导弹、友机、目标数量维度不能变。
2. 真实实体必须放在数组前部，后面补 0，并正确设置 mask。
3. `allocation_index` 必须结合 `allocation_mask` 使用。
4. 如果修改 `shape_config.py`，必须重新生成数据、重新训练、重新导出 ONNX。
5. 如果修改 `SituationAssessment.py`、`ThreatAssessment.py` 或 `TargetAllocation.py` 的规则公式，旧数据和旧模型都不再对应新规则，需要重新生成数据和训练。
6. `.pth` 权重加载时，训练和评估/导出的模型超参数必须一致，例如 `hidden_dim`、`d_model`、`num_heads`、`num_layers`、`ff_dim`。
7. 当前源码中的部分原中文注释已经出现乱码，已在代码旁补充英文说明，离线环境中优先参考英文注释和本文档。

## 7. 建议离线验证命令

进入 `Version1` 目录后运行：

```powershell
python test_version1_migration.py
python pipeline_test.py --onnx version1_aircombat_pipeline.onnx
```

如果离线环境未安装 `onnxruntime`，`pipeline_test.py` 会跳过 ONNX 推理部分；真正部署前应安装 `onnxruntime` 并确认 ONNX 推理能够执行。

## 8. 常用训练和导出命令

```powershell
python generate_dataset.py --samples 200000 --seed 42 --output aircombat_data_200k.npz
python train_situation.py --data aircombat_data_200k.npz --output situation_model_200k.pth --epochs 50 --batch-size 256
python train_threat.py --data aircombat_data_200k.npz --output threat_model_200k.pth --epochs 50 --batch-size 256
python train_allocation.py --data aircombat_data_200k.npz --situation-model situation_model_200k.pth --threat-model threat_model_200k.pth --output allocation_model_200k.pth --epochs 50 --batch-size 256
python evaluate_models.py --data aircombat_data_200k.npz --situation-model situation_model_200k.pth --threat-model threat_model_200k.pth --allocation-model allocation_model_200k.pth --batch-size 256
python export_onnx.py --situation-model situation_model_200k.pth --threat-model threat_model_200k.pth --allocation-model allocation_model_200k.pth --output-dir .
```
