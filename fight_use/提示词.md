请先检查当前工作目录父目录中的目标分配代码。现有算法只分配友方导弹攻击敌方飞机；请复用其输入接口、实体编号和可用结构，新增一个可独立运行的专家规则版本，使每枚友方导弹可以在“敌方飞机+敌方导弹”中统一选择目标。不要假设存在其他专家规则代码。

输入实体均为六维状态：
[x, y, z, vx, vy, vz]

输入包括：
友方导弹A、友方飞机F、敌方飞机X、敌方导弹Y。

核心原则：

综合分数 S[a,t] =
(T[t] + epsilon)^alpha
×
(G[a,t] + epsilon)^beta

T[t]表示敌目标t对友方飞机的全局威胁，G[a,t]表示友弹a处理目标t的交战有利度，二者均归一化到[0,1]。

默认：
alpha=1.2，beta=1.0，epsilon=1e-6，并放入配置文件。

一、相对运动计算

对任意源实体和目标实体计算：

相对位置 r = p_target - p_source
相对速度 v = v_target - v_source
当前距离 d = ||r||
接近速度 closing = max(0, -r·v / ||r||)
CPA时间 t_cpa = max(0, -r·v / ||v||²)
CPA距离 d_cpa = ||r + v*t_cpa||
方向余弦 cosine = source_velocity·r / (||source_velocity||*||r||)

使用exp、sigmoid、clamp等连续函数，将距离、CPA、方向、接近速度和时间归一化到[0,1]。敌机和敌弹允许使用不同归一化尺度，但最终分数语义必须一致。

二、威胁矩阵

计算：

enemy_plane_threat_matrix：F_max×X_max
enemy_missile_threat_matrix：F_max×Y_max

每个元素表示一个敌目标对一架友机的威胁，综合考虑：

当前距离、CPA距离、CPA时间、接近速度、飞行方向和时间紧迫度。

不能只看d_cpa，必须对很久以后才接近的目标进行时间衰减，避免极远目标得到接近1的威胁。

每个敌目标的全局威胁取其对所有真实友机威胁的最大值：

T_plane[x] = max_f threat_plane[f,x]
T_missile[y] = max_f threat_missile[f,y]

同时记录其主要威胁的友机编号和对应t_cpa。

三、交战有利度矩阵

计算：

attack_advantage_matrix：A_max×X_max
intercept_advantage_matrix：A_max×Y_max

攻击敌机的有利度综合：

距离、方向、相对速度、接近速度、CPA时间、CPA距离和高度差。

拦截敌弹除上述因素外，还要计算：

time_margin[a,y]
=
敌弹y接近其主要威胁友机的时间
-
友弹a接近敌弹y的时间

用：

time_advantage = sigmoid(time_margin / time_scale)

将时间裕度变成连续软评分，并纳入拦截有利度。

不要建立可达性硬约束。目标在后方、当前没有接近、CPA较大或时间裕度为负时，只降低G，不直接排除。唯一硬掩码是实体是否真实存在。

四、统一评分与分配

敌机评分：

S_plane[a,x]
=
(T_plane[x]+epsilon)^alpha
×
(G_attack[a,x]+epsilon)^beta

敌弹评分：

S_missile[a,y]
=
(T_missile[y]+epsilon)^alpha
×
(G_intercept[a,y]+epsilon)^beta

合并成：

combined_score_matrix：
A_max×(X_max+Y_max)

列顺序固定为：
前X_max列是敌机，后Y_max列是敌弹。

不要加入attack/defense倍率、攻防偏置或紧急防御覆盖。敌机和敌弹必须依靠统一尺度的T和G直接竞争。

每枚真实友弹在真实目标列中选择本行最高分目标。暂时允许多枚友弹选择同一目标，不设置目标容量，不做全局匹配。平分时选全局编号较小者。

五、矩阵与掩码

内部按照实际输入数量计算，对外补齐到配置中的固定最大维度，并输出：

friendly_missile_mask
friendly_plane_mask
enemy_plane_mask
enemy_missile_mask
target_mask

不存在的目标在展示矩阵中可填0，在用于argmax的masked_score_matrix中必须填INVALID_SCORE。

六、主要输出

输出至少包含：

1. 两个威胁矩阵及全局威胁向量；
2. 攻击敌机和拦截敌弹的有利度矩阵；
3. 敌弹拦截时间优势矩阵；
4. combined_score_matrix和masked_score_matrix；
5. assignment；
6. 简洁决策：友弹ID、ATTACK/INTERCEPT、目标类型、目标ID、威胁分、有利度、综合分；
7. 所有实体掩码。

请把公式、归一化尺度和权重集中到配置中，补充README和单元测试。测试必须覆盖矩阵维度、掩码、指数公式、时间衰减、敌机敌弹统一竞争、无硬可达性排除，以及多枚友弹选择同一目标。





# 第二版

请先检查当前目录父目录中的目标分配代码。现有算法只支持友方导弹攻击敌方飞机，请在其基础上新增一个独立的专家规则版本，让每枚友方导弹可以统一选择敌方飞机或敌方导弹。不要假设存在其他专家规则代码。

输入包括友方导弹、友方飞机、敌方飞机和敌方导弹，每个实体状态为位置和速度：[x,y,z,vx,vy,vz]。

算法分三步。

第一步，计算敌目标威胁。

分别计算：

1. 敌机对每架友机的威胁矩阵；
2. 敌弹对每架友机的威胁矩阵。

对每个“敌目标—友机”组合，根据双方位置和速度计算：

- 当前距离；
- 双方距离缩短速度；
- 敌目标是否朝向友机；
- 按当前速度直线运动时，多久后达到最近接近点；
- 最近接近时双方距离；
- 高度差。

目标越近、朝向友机、接近越快、最近接近时间越短、最近接近距离越小，威胁越高。所有因素用指数衰减、Sigmoid或截断函数平滑转换到0～1，不能使用突然跳变的阈值。

必须加入时间衰减，不能让距离很远、很久以后才接近的目标仅因为最近接近距离小就获得很高威胁。

敌机和敌弹可以使用不同的速度、距离和时间归一化参数，但最终威胁分数都必须在0～1之间，并具有相同含义。

每个敌目标的全局威胁，取它对所有真实友机威胁的最大值，同时记录它主要威胁哪架友机，以及预计最近接近时间。

第二步，计算友弹交战有利度。

分别计算：

1. 每枚友弹攻击每架敌机的有利度矩阵；
2. 每枚友弹拦截每枚敌弹的有利度矩阵。

根据友弹与目标的当前距离、方向关系、相对速度、距离缩短速度、最近接近时间、最近接近距离和高度差计算有利度。

友弹距离目标越近、方向越有利、接近越快、最近接近距离越小，有利度越高。

拦截敌弹时，还要比较两个时间：

- 敌弹还需要多久接近其主要威胁的友机；
- 友弹还需要多久接近该敌弹。

友弹越能提前接近敌弹，时间有利度越高；如果时间不够，只通过Sigmoid等连续函数降低分数，不直接判定不可拦截。

所有攻击和拦截有利度统一归一化到0～1。距离、方向和时间条件只作为软评分，不设置可达性硬约束。

第三步，融合威胁和有利度。

对每个“友弹—目标”组合，将目标威胁和友弹交战有利度综合：

综合分数 =
(目标威胁 + 很小的epsilon)的alpha次方
×
(交战有利度 + epsilon)的beta次方

默认：

alpha=1.2
beta=1.0
epsilon=0.000001

alpha越大越强调目标威胁，beta越大越强调友弹是否适合处理该目标。这三个参数必须放入配置文件。

不要使用攻防策略偏置、攻击倍率、防御倍率或紧急防御强制覆盖。敌机和敌弹必须依靠统一尺度的威胁和有利度直接竞争。

每枚真实友弹选择其综合分数最高的真实目标。暂时允许多枚友弹选择同一目标，不限制目标容量，不做全局匹配。分数相同时选择全局编号较小的目标。

内部按照实际实体数量计算，对外输出时补齐到配置规定的最大维度，并输出对应掩码。唯一硬限制是填充的不存在目标不能被选择。

输出应以矩阵为主，包括：

- 敌机对友机威胁矩阵；
- 敌弹对友机威胁矩阵；
- 敌机和敌弹的全局威胁；
- 友弹攻击敌机有利度矩阵；
- 友弹拦截敌弹有利度矩阵；
- 敌弹拦截时间有利度矩阵；
- 合并后的友弹—全部目标综合评分矩阵；
- 实体掩码和目标掩码；
- 每枚友弹的最终目标、动作类型、威胁分、有利度和综合分。

敌机列放在前面，敌弹列放在后面。请集中管理归一化尺度、权重、alpha和beta，更新示例、README和单元测试，确保极远期目标会被时间衰减，并验证敌机和敌弹能够在同一个综合评分矩阵中直接比较。
